1. JVM 基础

1.1 java 从编码到执行

  • 编译器编译源文件为 class 文件
  • java 运行时通过 classLoader 加载 class 文件到内存
  • JVM 解释执行 class 文件

1.2 JVM，从跨平台的语言到跨语言的平台

目前 JVM 支持任何能编译为 class 文件的语言（比如 kotlin、goovy、scala），支持多个操作系统。

JVM 根 java 没有什么必然关系，任何能编译为字节码文件的语言都可以使用 JVM 解释执行。

1.3 JVM 是一种规范

java virtual machine specifications 。

官方规范文档：

  • https://docs.oracle.com/en/java/javase/13/
  • https://docs.oracle.com/javas/specs/index.html

JVM 是虚构出来的一台计算机，它有自己的：

  • 字节码指令集（汇编语言）

  • 内存管理：栈、堆、方法区等

2.2.2 根据格式分析类文件

  • CAFEBABE

  • 0000 0034：jdk1.8

  • 0010：常量池中常量个数 16-1 = 15

  • 0a：第一个常量，tag 为 10，对应 CONSTANT_Methodref_info

    • 0003：该方法所在的类描述符 CONSTANT_Class_info 在常量池的 3 号位置。
    • 000d：该方法的名称以及类型描述符 CONSTANT_NameAndType 在常量池的 13 号位置

  • 07：第二个常量，tag 为 7，对应 CONSTANT_Class_info 该类的符号引用

    • 000e：在常量池的 14 号位置

  • 07：第三个常量，tag 为 7

    • 000f：在常量池的 15 号位置

  • 01：第四个常量，tag 为 1，对应 UTF-8 编码的字符串

    • 0006：UFT-8 编码的字符串占用了 6 个字节。
      • 3c69 6e69 743e：转为字符串就是 <init>

  • 01：第五个常量，tag 为 1

    • 0003：三个字节
      • 28 2956：转为字符串就是 ()V

  • 01：第五个常量，tag 为 1

    • 0004：四个字节
      • 43 6f64 65：转为字符串就是 Code

2.3 小结

.class字节码文件就是一堆十六进制数，按照 JVM 的规范排列在一起。JVM 根据规范依次解析出我们写的代码，然后转成机器码去和操作系统交互。

.class文件的构成本身不难，就是需要像查字典一样，一个一个去查询。
