    1、Seata Server 部署

    以类似生产环境形式部署 Seata Server 服务。模式使用 db，注册中心使用 nacos，配置中心也使用 nacos。

    1.1 前置条件

    • 安装 mysql：不做说明。见 mysql 相关第一篇笔记。

    • 安装 nacos：比较简单，下载安装文件，解压后直接运行即可。更多说明见nacoe 官方文档。

      # 下载 nacos 安装文件
https://github.com/alibaba/nacos/tags
https://github.com/alibaba/nacos/archive/1.3.2.tar.gz
# 解压安装包
tar -xvf nacos-server-1.3.2.tar.gz
# 启动 server 端
sh startup.sh -m standalone
# 关闭 server 端
sh shutdown.sh

# 注册服务
curl -X POST 'http://127.0.0.1:8848/nacos/v1/ns/instance serviceName=nacos.naming.serviceName&ip=20.18.7.10&port=8080'
# 服务发现
curl -X GET 'http://127.0.0.1:8848/nacos/v1/ns/instance/list?serviceName=nacos.naming.serviceName'
# 发布配置
curl -X POST "http://127.0.0.1:8848/nacos/v1/cs/configs dataId=nacos.cfg.dataId&group=test&content=HelloWorld"
# 获取配置
curl -X GET "http://127.0.0.1:8848/nacos/v1/cs/configs?dataId=nacos.cfg.dataId&group=test"

    • 安装 eureka：nacos 和 eureka 随便用哪个都行。

    1.2 下载并配置 Seata Server

    下载 seata

    直接去官网下载即可。https://github.com/seata/seata/archive/v1.3.0.tar.gz

    修改registry.conf配置

    registry.conf主要配置 seata 的注册中心和配置中心，相当于 seata 的基础配置，需要放在本地。其他配置可以放入注册中心即可。

    # 使用 eureka 作为注册中心
registry {
  # file,nacos,eureka,redis,zk,consule
  type = "eureka"

  nacos {
    application = "seata-server"
    serverAddr = "192.168.47.11:8848"
    group = "SEATA_GROUP"
    namespace = ""
    cluster = "default"
    username = "nacos"
    password = "nacos"
  }

  eureka {
    serviceUrl = "http://localhost:8761/eureka"
    # 指定seata服务名称
    application = "seata-server"
    weight = "1"
  }
}

# 使用 nacos 作为配置中心
config {
  type = "nacos"
  nacos {
    serverAddr = "192.168.47.11:8848"
    namespace = ""
    group = "SEATA_GROUP"
    username = "nacos"
    password = "nacos"
  }
}

    发布其他配置到 nacos

    在 config.txt 中写入配置，然后使用官方提供的发布脚本进行发布。

    1. 创建 config.txt 文件，写入如下配置。主要配置 seata 模式和数据源相关信息。
    store.mode=db
store.db.datasource=druid
store.db.dbType=mysql
store.db.driverClassName=com.mysql.jdbc.Driver
store.db.url=jdbc:mysql://192.168.47.11:3306/seata-server?useUnicode=true&useSSL=false&characterEncoding=utf8&serverTimezone=Asia/Shanghai
store.db.user=root
store.db.password=pass9876
store.db.minConn=5
store.db.maxConn=30
store.db.globalTable=global_table
store.db.branchTable=branch_table
store.db.queryLimit=100
store.db.lockTable=lock_table
store.db.maxWait=5000

    1. 然后执行 sh nacos-config.sh 命令发布配置。

    2. 发布完成即可在 nacos 控制台看到刚刚发布的配置。

      http://192.168.47.11:8848/nacos 用户名/密码：nacos/nacos

    nacos-config.sh 脚本获取：https://github.com/seata/seata/tree/develop/script/config-center/nacos/nacos-config.sh

    配置数据库

    db 模式需要创建几个数据表用来给 seata server 使用。

    -- 创建数据库
SHOW CHARACTER SET;
SHOW COLLATION;
CREATE DATABASE `seata-server` CHARACTER SET = utf8mb4 COLLATE = utf8mb4_general_ci;

-- 分支事务表
CREATE TABLE `branch_table` (
  `branch_id` bigint(20) NOT NULL,
  `xid` varchar(128) CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL,
  `transaction_id` bigint(20) DEFAULT NULL,
  `resource_group_id` varchar(32) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `resource_id` varchar(256) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `branch_type` varchar(8) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `status` tinyint(4) DEFAULT NULL,
  `client_id` varchar(64) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `application_data` varchar(2000) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `gmt_create` datetime DEFAULT NULL,
  `gmt_modified` datetime DEFAULT NULL,
  PRIMARY KEY (`branch_id`) USING BTREE,
  KEY `idx_xid` (`xid`) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8 ROW_FORMAT=DYNAMIC;

-- 全局事务表
CREATE TABLE `global_table` (
  `xid` varchar(128) CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL,
  `transaction_id` bigint(20) DEFAULT NULL,
  `status` tinyint(4) NOT NULL,
  `application_id` varchar(32) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `transaction_service_group` varchar(32) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `transaction_name` varchar(128) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `timeout` int(11) DEFAULT NULL,
  `begin_time` bigint(20) DEFAULT NULL,
  `application_data` varchar(2000) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `gmt_create` datetime DEFAULT NULL,
  `gmt_modified` datetime DEFAULT NULL,
  PRIMARY KEY (`xid`) USING BTREE,
  KEY `idx_gmt_modified_status` (`gmt_modified`,`status`) USING BTREE,
  KEY `idx_transaction_id` (`transaction_id`) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8 ROW_FORMAT=DYNAMIC;

-- 全局锁
CREATE TABLE `lock_table` (
  `row_key` varchar(128) CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL,
  `xid` varchar(96) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `transaction_id` bigint(20) DEFAULT NULL,
  `branch_id` bigint(20) NOT NULL,
  `resource_id` varchar(256) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `table_name` varchar(32) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `pk` varchar(36) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  `gmt_create` datetime DEFAULT NULL,
  `gmt_modified` datetime DEFAULT NULL,
  PRIMARY KEY (`row_key`) USING BTREE,
  KEY `idx_branch_id` (`branch_id`) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8 ROW_FORMAT=DYNAMIC;

    1.3 启动 seata server

    启动服务：

    nohup sh seata-server.sh -h 192.168.47.11 -p 8091 -m db -n 1 &
    参数 全写 作用 备注
    -h –host 指定在注册中心注册的 IP 不指定时获取当前的 IP，外部访问部署在云环境和容器中的 server 建议指定
    -p –port 指定 server 启动的端口 默认为 8091
    -m –storeMode 事务日志存储方式 支持file,db,redis，默认为 file 注:redis需seata-server 1.3版本及以上
    -n -serverNode 用于指定seata-server节点ID 如 1,2,3…, 默认为 1
    -e –seataEnv 指定 seata-server 运行环境 如 dev, test 等, 服务启动时会使用 registry-dev.conf 这样的配置

    2.1 创建测试数据库

    -- 创建三个数据库
CREATE DATABASE `seate-rm-one` CHARACTER SET = utf8mb4 COLLATE = utf8mb4_general_ci;
CREATE DATABASE `seate-rm-two` CHARACTER SET = utf8mb4 COLLATE = utf8mb4_general_ci;
CREATE DATABASE `seate-rm-three` CHARACTER SET = utf8mb4 COLLATE = utf8mb4_general_ci;

-- 创建业务表
DROP TABLE IF EXISTS `seate-rm-one`.tbl_one;
DROP TABLE IF EXISTS `seate-rm-two`.tbl_two;
DROP TABLE IF EXISTS `seate-rm-three`.tbl_three;

CREATE TABLE `seate-rm-one`.tbl_one(id INT PRIMARY KEY AUTO_INCREMENT, NAME VARCHAR(50)) ENGINE INNODB DEFAULT CHARSET utf8;
CREATE TABLE `seate-rm-two`.tbl_two(id INT PRIMARY KEY AUTO_INCREMENT, NAME VARCHAR(50)) ENGINE INNODB DEFAULT CHARSET utf8;
CREATE TABLE `seate-rm-three`.tbl_three(id INT PRIMARY KEY AUTO_INCREMENT, NAME VARCHAR(50)) ENGINE INNODB DEFAULT CHARSET utf8;

SELECT * FROM `seate-rm-one`.tbl_one;
SELECT * FROM `seate-rm-two`.tbl_two;
SELECT * FROM `seate-rm-three`.tbl_three;

-- 三个库均创建 undo_log 表，用于 RM 回滚
CREATE TABLE `undo_log` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT,
  `branch_id` bigint(20) NOT NULL,
  `xid` varchar(100) CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL,
  `context` varchar(128) CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL,
  `rollback_info` longblob NOT NULL,
  `log_status` int(11) NOT NULL,
  `log_created` datetime NOT NULL,
  `log_modified` datetime NOT NULL,
  `ext` varchar(100) CHARACTER SET utf8 COLLATE utf8_general_ci DEFAULT NULL,
  PRIMARY KEY (`id`) USING BTREE,
  UNIQUE KEY `ux_undo_log` (`xid`,`branch_id`) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8 ROW_FORMAT=DYNAMIC;

    2.4 PhaseTwo_RollbackFailed_Retryable

    回滚时报如下错误：

    Field not equals, name NAME, old value rm2, new value 333
branchRollback failed. branchType:[AT], xid:[192.168.47.11:8091:102850563426750464]
Branch Rollbacked result: PhaseTwo_RollbackFailed_Retryable

    场景还原：

    1. 在全局事务回滚前（报错前），修改已经提交的局部事务。这里是在插入seate-rm-two之后，插入seate-rm-three之前断点，然后手动去数据库把值由 rm2 修改为 333
    2. 放开断点，由于 undo_log 中的数据和当前数据库中数据不一致，导致一直回滚失败。

    如果有人在事务回滚期间修改了相关数据，那么有可能导致回滚日志一直未能成功？

    答：所以at需要通过全局锁来修改数据，如果没拿到锁是不允许的，也就是如果有人直接在回滚前去数据库修改了，就会把数据变脏
