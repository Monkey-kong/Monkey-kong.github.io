    分布式配置管理之disconf

    1. disconf-web

    disconf 有两个模块，disonf-web 就是分布式配置中心服务端，disconf-client 就是客户端，服务端搭建好后，需要接入 disconf 管理配置的应用引入 disconf-client 就行。

    1.1 安装 disconf-web

    安装依赖软件

    • MySQL：MySQL 安装教程
    • Tomcat：官网下载后解压即可
    • Nginx：OpenResty 安装教程
    • zookeeper：zookeeper 安装教程
    • Redis：Redis 安装教程

    下载 disconf

    下载：https://github.com/knightliao/disconf/archive/refs/tags/2.6.36.zip

    编译打包

    -- 软件打包目录创建
mkdir -p /home/disconf/online-resources
mkdir -p /home/disconf/war
-- 进入 disconf-web 源码目录进行打包
cd disconf-master/disconf-web
mvn clean install -Dmaven.javadoc.skip=true -Dmaven.test.skip=true

    修改配置

    -- 拷贝/disconf-web/profile/rd/下的配置到 /home/disconf/online-resources
cp /disonf-web/profile/rd/* /home/disconf/online-resources
-- 创建 application.properties 配置文件
cp application-demo.properties application.properties

-- 修改配置，按照自己实际情况修改。主要修改一些连接信息
jdbc-mysql.properties (数据库配置)
-- 即使只有一个redis，也应该配置两个redis client，否则将造成内部错误。

export ONLINE_CONFIG_PATH
export WAR_ROOT_PATH
-- 执行disonf-web下的部署脚本
sh deploy/deploy.sh

    1.2 遇到的问题

    实例列表和zk部署情况都为空

    后台检查 zk 服务是正常的，zkServer.sh status 都正常，但是 discon-web 却查不到 zk 部署情况，发现可能是 disconf-web 的依赖版本和 zk 服务版本不一致。所以修改 disconf-web 依赖的 zk 版本。

    修改这个版本后发现启动 tomcat 时，disconf-web war 包并没有成功启动，看 tomcat 日志，发现是 sl4j 日志包版本有问题，新版本的 zk 中依赖的 sl4j 版本更高，所以需要把 sl4j 版本也升高，查看 3.7.0 zk sl4j 版本，发现是1.7.30。

    修改完成后，重新编译打包 disconf-web，然后部署，就正常了。

    2. disconf-client

    使用一个简单的demo项目来验证 disconf-client 相关功能。demo 项目就是每过 5 秒钟读取一次 redis 的一个 key，并打印结果。

    redis.proserties

    redis.host=192.168.47.11
redis.port=6379
redis.password=pass9876

    JedisConfig

    @Service
@Scope("singleton")
public class JedisConfig {
    private String host;
    private int port;
    private String password;
    // 省略 getter/setter
}

    SimpleRedisService

    @Service
@Scope("singleton")
public class SimpleRedisService implements InitializingBean, DisposableBean {

    protected static final Logger LOGGER = LoggerFactory
            .getLogger(SimpleRedisService.class);

    private Jedis jedis = null;

    @Autowired
    private JedisConfig jedisConfig;

    @Override
    public void destroy() throws Exception {
        if (jedis != null) {
            jedis.disconnect();
        }
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        jedis = JedisUtil.createJedis(jedisConfig.getHost(),
                jedisConfig.getPort(), jedisConfig.getPassword());
    }

    public String getKey(String key) {
        if (jedis != null) {
            return jedis.get(key);
        }
        return null;
    }
}

    JedisUtil

    public class JedisUtil {
    public static Jedis createJedis(String host, int port, String password) {
        Jedis jedis = new Jedis(host, port);
        if (!StringUtils.isNotBlank(password)) {
            jedis.auth(password);
        }
        return jedis;
    }
}

    DisconfDemoTask

    @Service
public class DisconfDemoTask {

    protected static final Logger LOGGER = LoggerFactory
            .getLogger(DisconfDemoTask.class);

    @Autowired
    private SimpleRedisService simpleRedisService;

    @Autowired
    private JedisConfig jedisConfig;

    private static final String REDIS_KEY = "disconf_key";

    public int run() {
        try {
            while (true) {
                Thread.sleep(5000);
                LOGGER.info("redis( " + jedisConfig.getHost() + ","
                        + jedisConfig.getPort() + ")  get key: " + simpleRedisService.getKey(REDIS_KEY));
            }
        } catch (Exception e) {
            LOGGER.error(e.toString(), e);
        }
        return 0;
    }
}

    添加Disconf的支持

    在applicationContext.xml里添加Bean定义：

    修改 JedisConfig 支持分布式配置

    类上面加DisconfFile注解。get 方法上加 DisconfFileItem 注解。

    @Service
@Scope("singleton")
@DisconfFile(filename = "redis.properties")
public class JedisConfig {

    private String host;
    private int port;
    private String password;

    @DisconfFileItem(name = "redis.host")
    public String getHost() {
        return host;
    }

    public void setHost(String host) {
        this.host = host;
    }

    @DisconfFileItem(name = "redis.port")
    public int getPort() {
        return port;
    }

    public void setPort(int port) {
        this.port = port;
    }

    @DisconfFileItem(name = "redis.password")
    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }
}

    2.2 配置更新通知

    比如有一天需要连接另外的 Redis 服务器，那么就需要修改 Redis 的 Host 和 Port，然后重启服务。但是 disconf 可以实现配置动态更新，只需修改配置，无需重启服务。注意：Host、Port 的值随着配置的修改已经修改，但是 Jedis 连接需要手动写方法去重新构建。

    @Service
@DisconfUpdateService(classes = {JedisConfig.class})
public class SimpleRedisServiceUpdateCallback implements IDisconfUpdate {

    private static final Logger LOGGER = LoggerFactory.getLogger(SimpleRedisServiceUpdateCallback.class);

    @Autowired
    private JedisConfig jedisConfig;

    @Autowired
    private SimpleRedisService simpleRedisService;

    @Override
    public void reload() throws Exception {
        LOGGER.info("JedisConfig 有配置更新！host:{},port:{},passeword:{}", jedisConfig.getHost(), jedisConfig.getPort(), jedisConfig.getPassword());
        // simpleRedisService.changeJedis();
    }
}

    2.3 注解式分布式配置项

    有时我们只需要一个单独的配置项，不想创建文件，disconf 也支持分布式配置项。

    比如之前例子是每五秒钟从 Redis 读取一次 disconf_key 这个 Redis key 的值，现在把这个 Redis key 作为配置动态化。

    只需在配置项 get 方法上加上 @DisconfItem(key = DYNAMIC_CONFIG_KEY) 注解，然后配置配置中心创建对应配置即可。

    @Service
public class DisconfDemoTask {

    protected static final Logger LOGGER = LoggerFactory
            .getLogger(DisconfDemoTask.class);

    @Autowired
    private SimpleRedisService simpleRedisService;

    @Autowired
    private JedisConfig jedisConfig;

    // 分布式配置项的key
    private static final String DYNAMIC_CONFIG_KEY = "config_item";

    // redis key。具体值从分布式配置项中获取。
    // 使用 @Value 给个默认值
    @Value("disconf_key")
    private String reidsKey;

    public int run() {
        try {
            while (true) {
                Thread.sleep(5000);
                LOGGER.info("redis( " + jedisConfig.getHost() + ","
                        + jedisConfig.getPort() + ")  current key: " + getReidsKey()
                        + "，current value:" + simpleRedisService.getKey(getReidsKey()));
            }
        } catch (Exception e) {
            LOGGER.error(e.toString(), e);
        }
        return 0;
    }

    // 获取动态配置
    @DisconfItem(key = DYNAMIC_CONFIG_KEY)
    public String getReidsKey() {
        return reidsKey;
    }

    public void setReidsKey(String reidsKey) {
        this.reidsKey = reidsKey;
    }
}

    分布式配置项也可以监听更新。

    @DisconfUpdateService(classes = {JedisConfig.class}, itemKeys = {DisconfDemoTask.DYNAMIC_CONFIG_KEY})

    2.4 分布式配置文件（非properties）

    前面的例子都是.properties 文件或者直接配置项，这样可以用 java 实例进行映射，然后通过 disconf 提供的注解实现自动更新。但是如果不是 properties 就没法实现自动更新，这时可以使用 xml 形式配置文件的托管，应用启动 disconf 会把文件下载下载，文件修改后会自动更新配置文件。

    <!-- 使用托管方式的disconf配置(无代码侵入, 配置更改不会自动reload)-->
<bean id="configproperties_no_reloadable_disconf" class="com.baidu.disconf.client.addons.properties.ReloadablePropertiesFactoryBean">
    <property name="locations">
        <list>
            <value>myserver.properties</value>
        </list>
    </property>
</bean>

<bean id="propertyConfigurerForProject1" class="org.springframework.beans.factory.config.PropertyPlaceholderConfigurer">
    <property name="ignoreResourceNotFound" value="true"/>
    <property name="ignoreUnresolvablePlaceholders" value="true"/>
    <property name="propertiesArray">
        <list>
            <ref bean="configproperties_no_reloadable_disconf"/>
        </list>
    </property>
</bean>

    2.5 正真意义的统一线上包

    我们发现使用 disconf 本身需要一个配置文件disconf.properties，对于不同环境还是需要不同的 disconf 配置文件。为了解决这个问题，disconf 支持 Java 启动参数形式进行 disconf 的相关配置。

    java  -Ddisconf.env=rd \
    -Ddisconf.enable.remote.conf=true \
    -Ddisconf.conf_server_host=127.0.0.1:8000 \
    -Dlogback.configurationFile=logback.xml \
    -Dlog4j.configuration=file:log4j.properties \
    -Djava.ext.dirs=lib \
    -Xms1g -Xmx2g -cp ampq-logback-client-0.0.1-SNAPSHOT.jar \
    com.github.knightliao.consumer.ConsumerMain >/dev/null 2>&1 &

    2.6 配置文件下载位置

    disconf.user_define_download_dir

    配置了该配置，则肯定会下载到这个目录，并且配置更新该目录的文件也会更新。

    disconf.enable_local_download_dir_in_class_path

    该配置默认为 true，即默认也会把文件下载到 classpath 目录，如果不想下载到 classpath，设置为 false 即可。

    代码配置默认从 classpath 取，classpath 没有（设置为 false）则从自定义目录取。
