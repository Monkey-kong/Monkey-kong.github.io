    vuex 状态管理教程

    1. 安装

    https://unpkg.com/vuex

    直接在引入 vue 之后引入 vuex

    <script src="/path/to/vue.js"></script>
<script src="/path/to/vuex.js"></script>

    在模块化打包系统中，必须显示地通过 Vue.use()来安装 Vuex

    npm install vuex --save
yarn add vuex

import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

    2. Vuex 是什么?

    Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。

    当我们的应用遇到多个组件共享状态时，单向数据流的简洁性很容易被破坏：

    • 多个视图依赖于同一状态

    • 来自不同视图的行为需要变更同一状态。

      对于问题一，传参的方法对于多层嵌套的组件将会非常繁琐，并且对于兄弟组件间的状态传递无能为力。

      对于问题二，我们经常会采用父子组件直接引用或者通过事件来变更和同步状态的多份拷贝。以上的这些模式非常脆弱，通常会导致无法维护的代码。

    总的来说就是使用 Vuex 使得状态的管理变得简单，不用各个组件间传来传去，优点类似全局变量和缓存的意思，但是 Vuex 中的状态是响应式的，轻松实现数据双向绑定；并且状态的改变只能显示 commit，方便跟踪状态变化。

    vue 单个组件 data 单向数据流

    Vuex 多个组件状态管理

    3. 开始

    每一个 Vuex 应用的核心就是 store（仓库）。“store”基本上就是一个容器，它包含着你的应用中大部分的**状态 (state)**。Vuex 和单纯的全局对象有以下两点不同：

    1. Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。
    2. 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。这样使得我们可以方便地跟踪每一个状态的变化 
    <script>
    // 定义 store
const store = new Vuex.Store({
    // store 中的状态
  state: {
    count: 0
  },
  // 修改状态的方法
  mutations: {
    increment (state) {
      state.count++
    }
  }
});

// 修改状态
store.commit('increment')
store.commit('increment')
console.log(store.state.count) // -> 2

</script>

    为了在 Vue 组件中访问 this.$store property，你需要为 Vue 实例提供创建好的 store。 

    var vm = new Vue({
    el: "#app",
    data: {
    },
    methods: {
        increment() {
            this.$store.commit('increment')
            // js 中通过 this.$store 访问状态
            console.log(this.$store.state.count)
        }
    },
    store: store,
    computed: {

    }
});

    3. 核心概念

    3.1 State

    单一状态树

    每个应用将仅仅包含一个 store 实例。

    在 Vue 组件中获得 Vuex 状态

    1. 全局变量形式

      组件依赖全局状态单例。 在模块化的构建系统中，在每个需要使用 state 的组件中需要频繁地导入，并且在测试组件时需要模拟状态。 

    <body>
    <div id="app">
        <div id="app">
            <h1>{{count}}</h1>
            <button @click="increment">count+1</button>
        </div>
    </div>
</body>
<script>
    const store = new Vuex.Store({
        state: {
            count: 0
        },
        mutations: {
            increment(state){
                state.count++;
            }
        }
    });
    var vm = new Vue({
        el: "#app",
        computed: {
            // 计算属性实现绑定 state
            count(){
                return store.state.count;
            }
        },
        methods: {
            increment() {
                store.commit('increment')
            }
        }
    });
</script>

    然而，这种模式导致组件依赖全局状态单例。在模块化的构建系统中，在每个需要使用 state 的组件中需要频繁地导入，并且在测试组件时需要模拟状态。

    1. 直接添加 store 选项

      Vuex 通过 store 选项，提供了一种机制将状态从根组件“注入”到每一个子组件中（需调用 Vue.use(Vuex)）：

    var vm = new Vue({
    el: "#app",
    computed: {
        // 计算属性实现绑定 state
        count(){
            return this.$store.state.count;
        }
    },
    methods: {
        increment() {
            this.$store.commit('increment')
        }
    },
    // 这种形式会注册到所有子组件
    store
});

    mapState 辅助函数

    当一个组件需要获取多个状态的时候，将这些状态都声明为计算属性会有些重复和冗余。为了解决这个问题，我们可以使用 mapState 辅助函数和对象展开运算符帮助我们生成计算属性，让你少按几次键：

    就是简化this.$store.state.count为state.count

    computed: {
    // 计算属性实现绑定 state
    // count(){
    //     return this.$store.state.count;
    // }

    // 映射 this.count 为 store.state.count
    // 名字相同时，可以直接用数组
    // ...Vuex.mapState(['count'])
    // 不同时用对象
    ...Vuex.mapState({
        count: state => state.count,
        // 啥意思？
        countAlias: 'count',
        // 要使用 this 访问局部变量时，必须使用常规函数
        countPlusLocalState(state) {
            return state.count + this.localCount
        }
    })
},

    3.2 Getter

    有时候我们需要从 store 中现有的 state 中派生出一些新的状态。

    Vuex 允许我们在 store 中定义“getter”（可以认为是 store 的计算属性）。就像计算属性一样，getter 的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。 

    <body>
    <div id="app">
        已完成任务列表:{{doneTodos}}<hr>
        已完成任务个数:<h1>{{doneTodosCount}}</h1><hr>
        通过ID获取任务:{{getTodoById(2)}}
    </div>
</body>
<script>
// 定义状态库    
const store = new Vuex.Store({
    state: {
        // 所有任务列表
        todos: [
            { id: 1, text: '...', done: false },
            { id: 2, text: '...', done: true },
            { id: 3, text: '...', done: true }
        ]        
    },
    // 类似 vue 的 computed
    getters: {
        // 已处理列表
        doneTodos: state => {
            return state.todos.filter(todo => todo.done)
        },
        // 已处理个数
        doneTodosCount: (state, getters) => {
            return getters.doneTodos.length
        },
        // 返回一个函数
        getTodoById: (state) => (id) => {
            return state.todos.find(todo => todo.id === id)
        }                
    }
});

var vm = new Vue({
        el: "#app",
        computed: {
            // doneTodosCount () {
            //     // 通过属性访问状态
            //     return this.$store.getters.doneTodosCount
            // },
            ...Vuex.mapGetters(['doneTodosCount','doneTodos','getTodoById'])
        },
        store
    });
</script>

    3.3 Mutation

    每个 mutation 都有一个字符串的 事件类型 (type) 和 一个 **回调函数 (handler)**。这个回调函数就是我们实际进行状态更改的地方，并且它会接受 state 作为第一个参数：

    定义 mutations

    const store = new Vuex.Store({
  state: {
    count: 1
  },
  mutations: {
    increment (state) {
      // 变更状态
      state.count++
    }
  }
})

    唤醒 mutations

    // 唤醒 mutations handler
store.commit('increment')

    提交载荷（Payload）

    mutations: {
  increment (state, payload) {
    state.count += payload.amount
  }
}

store.commit('increment', {
  amount: 10
})

    对象风格的提交方式

    提交 mutation 的另一种方式是直接使用包含 type 属性的对象： 

    store.commit({
  type: 'increment',
  amount: 10
})

// handler 不变
mutations: {
  increment (state, payload) {
    state.count += payload.amount
  }
}

    Mutation 需遵守 Vue 的响应规则

    1. 最好提前在你的 store 中初始化好所有所需属性。

    2. 当需要在对象上添加新属性时，你应该

      1. 使用 Vue.set(obj, 'newProp', 123), 或者

      2. 以新对象替换老对象。state.obj = { …state.obj, newProp: 123 }

    Mutation 必须是同步函数

    因为异步函数不知道啥时候成功，导致状态无法被追踪。

    可以是使用this.$store.commit('xxx')在组件中提交 Mutation； 或者使用 mapMutations 辅助函数将组件中的 methods 映射为 store.commit 调用（需要在根节点注入 store）。 

    methods: {
    // 名字相同
    ...mapMutations([
        'increment',
        'incrementBy'
    ]),
    // 名字不相同    
    ...mapMutations({
        add: 'increment'
    })
}

    3.4 Actioin

    Action 类似于 mutation，不同在于：

    • Action 提交的是 mutation，而不是直接变更状态。
    • Action 可以包含任意异步操作。

    定义 Action

    actions: {
    increment(context){
        context.commit('increment')
    }
}

    分发 Action

    store.dispatch('increment')

    Action 同样支持以载荷形式分发和以对象形式分发。

    在组件中分发 Action

    你在组件中使用 this.$store.dispatch('xxx') 分发 action，或者使用 mapActions 辅助函数将组件的 methods 映射为 store.dispatch 调用（需要先在根节点注入 store）： 

    <body>
    <div id="app">
        <h1>{{$store.state.count}}</h1>
        <button @click="increment">dispatchByNormal</button>
        <button @click="incrementAsync">incrementAsync</button>
    </div>
</body>
<script type="module">
const store = new Vuex.Store({
    state: {
        count: 0
    },
    mutations: {
        increment(state){
            state.count++
        }
    },
    actions: {
        // 定义 action；context 是 store 对象的一个副本
        increment(context){
            context.commit('increment')
        },
        // action 可以执行异步操作
        incrementAsync({commit}){
            setTimeout(()=>{
                commit('increment')
            }, 1000)
        }
    }
});

// import { mapActions } from 'vuex'
var vm = new Vue({
    el: "#app",
    methods: {
        // dispatchByNormal(){
        //     this.$store.dispatch('incrementAsync');
        // },
        ...Vuex.mapActions(['increment','incrementAsync'])
    },
    store
})
</script>

    3.5 Module

    Vuex 允许我们将 store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块——从上至下进行同样方式的分割： 

    const moduleA = {
  state: () => ({ ... }),
  mutations: { 
    increment (state) {
      // 这里的 `state` 对象是模块的局部状态
      state.count++
    }  
  },
  actions: {
    // 模块内 action 通过 context.rootState 获取根节点状态 
    incrementIfOddOnRootSum ({ state, commit, rootState }) {
      if ((state.count + rootState.count) % 2 === 1) {
        commit('increment')
      }
    }  
  },
  getters: { 
    // 这里的 `state` 对象是模块的局部状态   
    doubleCount (state) {
      return state.count * 2
    },
    sumWithRootCount (state, getters, rootState) {
      return state.count + rootState.count
    }        
  }
}

const moduleB = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}

const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

    模块的局部状态

    对于模块内部的 mutation 和 getter，接收的第一个参数是模块的局部状态对象。

    同样，对于模块内部的 action，局部状态通过 context.state 暴露出来，根节点状态则为 context.rootState

    对于模块内部的 getter，根节点状态会作为第三个参数暴露出来

    命名空间

    默认情况下，模块内部的 action、mutation 和 getter 是注册在全局命名空间的——这样使得多个模块能够对同一 mutation 或 action 作出响应。

    如果希望你的模块具有更高的封装度和复用性，你可以通过添加 namespaced: true 的方式使其成为带命名空间的模块。
