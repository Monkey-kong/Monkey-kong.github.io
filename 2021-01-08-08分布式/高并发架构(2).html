    1、Redis 与 Lua 整合

    1.1 Redis 执行 Lua 脚本

    通过 Lua 脚本的全局变量 KEYS 和 ARGV，能够访问 EVAL 和 EVALSHA 命令的 key [key …] 参数和 arg [arg …] 参数。作为 Lua Table，能够将 KEYS 和 ARGV 作为一维数组使用，其下标从 1 开始。

    # 命令 脚本 参数个数
# 进入 redis 客户端执行
eval "return 1+1" 0
eval "local msg='hello world' return msg..KEYS[1]" 1 AAA BBB

# 直接在外部执行
redis-cli -p 6380 eval "return 1+1" 0

# 一个参数，取出参数的 key 和 value
redis-cli -p 6380 eval "local msg='hello:' return msg..KEYS[1]..'-'..ARGV[1]" 1 name alvin

    1.2 Redis 执行 Lua 脚本文件

    Lua 脚本内部允许通过内置函数redis.call()、redis.pcall()执行 Redis 命令

    • redis.call()：若 Redis 命令执行错误，将错误抛出
    • redis.pcall()：若 Redis 命令执行错误，将错误返回

    获取指定 key 的 value

    脚本文件内容：

    local key = KEYS[1]
local list = redis.call("get", key)
return list

    执行脚本：

    # 获取 count 这个 key 的 value
redis-cli -p 6380 --eval Test03.lua count

    获取集合

    脚本文件内容：

    local key=KEYS[1]
local list=redis.call("lrange",key,0,-1)
return list

    执行脚本：

    redis-cli -p 6380 --eval Test03.lua listkey

    网站计数

    脚本文件内容：

    local count = redis.call("get", "count")
redis.call("incr", "count")
return count

    执行脚本：

    redis-cli -p 6380 --eval Test03.lua

    远程执行

    redis-cli -h 192.168.2.161 -a 密码 --eval /usr/local/luascript/test.lua name age , xiao6

    1.3 Redis WATCH/MULTI/EXEC 与 Lua

    Redis 原生支持监听、事务、批处理，那么还需要 Lua 吗？

    两者不存在竞争关系，而是增强关系，lua 可以完成 redis 自身没有的功能。

    • lua 中可以使用上一步的结果，也就是可以后面的操作依赖前面操作的执行结果，MULTI 中的命令都是独立操作。
    • redis 本身也可以编写模块增强功能，但是 c 语言写模块，难度太大了，lua 简单的多
    • 计算向数据移动，类似 sql 语句或者存储过程
    • 原子操作

    lua 脚本要尽量短小且尽量保证同一事务写在一段脚本中，因为 redis 是单线程的，过长的执行会造成阻塞，影响服务器的性能。

    1.4 Redis lua 脚本管理

    1. script load：将 lua 脚本加载到 redis 内存中
    2. script exists sha1：判断 sha1 是否已经加载到 redis 内存中
    3. script flush：清除 redis 内存已经加载的所有 lua 脚本
    4. script kill：杀掉正在执行的 lua 脚本

    1.5 死锁

    下面代码会进入死循环，导致 redis 无法接受其他命令

    eval "while true do end" 0

    无法接受其他命令：

    E:\01_code\08_jg\lua>redis-cli -p 6380 eval "return 1+1" 0
(error) BUSY Redis is busy running a script. You can only call SCRIPT KILL or SHUTDOWN NOSAVE.

    但是可以接受 script kill 或者 shutdown nosave 两个命令。

    • shutdown nosave：不会进行持久化操作
    • script kill：杀死正在执行的进程
    E:\01_code\08_jg\lua>redis-cli -p 6380 script kill

E:\01_code\08_jg\lua>redis-cli -p 6380 eval "while true do end" 0
(error) ERR Error running script (call to f_694a5fe1ddb97a4c6a1bf299d9537c7d3d0f84e7): @user_script:1: Script killed by user with SCRIPT KILL...

    1.6 生产环境下部署 lua 脚本

    加载到 redis

    redis-cli script load "$(cat test.lua)"

    得到 sha1 值。

    执行

    redis-cli evalsha "7a2054836e94e19da22c13f160bd987fbc9ef146" 0

    2、Openresty Nginx + Lua

    Nginx 是一个主进程配合多个工作进程的工作模式，每个进程由单个线程来处理多个连接。在生产环境中，我们往往会把 cpu 内核直接绑定到工作进程上，从而提升性能。

    2.1 安装 Openresty

    官网安装说明：http://openresty.org/cn/installation.html

    centos 安装方式：

    预编译安装

    yum install yum-utils
yum-config-manager --add-repo https://openresty.org/package/centos/openresty.repo
yum install openresty
sudo yum install openresty-resty

    源码编译安装

    # 安装依赖
yum install gcc openssl-devel pcre-devel zlib-devel postgresql-devel
# 编译
./configure --prefix=/opt/openresty \
            --with-luajit \
            --without-http_redis2_module \
            --with-http_iconv_module \
            --with-http_postgres_module
# 查看更多编译选项
./configure --help
# 安装
make && make install

    服务命令

    service openresty start
service openresty stop
service openresty reload
service openresty status
# 检查配置文件是否正确
nginx -t
# 查看已安装模块和版本号
nginx -v

    2.2 Openresty 测试 lua 脚本

    # nginx.conf 中配置
location /lua {
    default_type text/html;
    content_by_lua '
        ngx.say("<h1>Hello, World!</h1>")
    ';
}

    访问：http://192.168.47.11/lua

    中文乱码解决：在 server 对象下配置 charset utf-8;

    2.3 Openresty 测试 lua 脚本文件

    # nginx.conf 中配置
location /lua {
    default_type text/html;
    content_by_lua_file conf/lua/hello.lua;
}

# 新建文件：/usr/local/openresty/nginx/conf/lua/hello.lua
ngx.say("<h1>Hello world!</h1>");

    2.5 获取 uri 中的所有变量

    ngx.say("获取 nginx uri 中的所有变量：", "<br/>");
local uri_args = ngx.req.get_uri_args();
for k,v in pairs(uri_args) do
    if type(v) == "table" then
        ngx.say(k, " : ", table.concat(v, ","), "<br/>");
    else
        ngx.say(k, " : ", v, "<br/>");
    end;
end;

    2.6 获取nginx请求头信息

    ngx.say("获取 nginx 请求头信息：", "<br/>");
local headers = ngx.req.get_headers();
ngx.say("Host:", headers["Host"], "<br/>");
ngx.say("user-agent:", headers["user-agent"], "<br/>");
ngx.say("user-agent:", headers.user_agent, "<br/>");

for k,v in pairs(headers) do
    if type(v) == "table" then
        ngx.say(k, " : ", table.concat(v, ","), "<br/>");
    else
        ngx.say(k, " : ", v, "<br/>");
    end;
end;

    2.7 获取 post 请求参数

    ngx.say("获取 post 请求参数：", "<br/>");

for k,v in pairs(post_args) do

        ngx.say(k, " : ", table.concat(v, ","), "<br/>");

        ngx.say(k, ":", v, "<br/>");
    end;
end;
