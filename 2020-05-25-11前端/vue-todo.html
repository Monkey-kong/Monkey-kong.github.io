import Vue from 'vue'

    为啥需要打包？

    因为有些语法和代码浏览器不认识，需要打包工具翻译一下。

    三. wepack 基本配置

    为了让浏览器能够正确的解析, 我们需要使用 webpack 将我们的源代码进行打包

    1. 创建 webpack 配置文件

    在根目录下创建 webpack 配置文件 webpack.config.js, 编写最基本的配置

    // 使用 node 的 path 模块
const path = require('path')
module.exports = {
    // 打包的入口
    entry: './src/main.js',
    // 打包的出口
    output: {
        filename: 'bundle.js',
        // resolve? __dirname??
        path: path.resolve(__dirname, 'dist')
    }
}
    • entry:指定打包的入口
    • output:指定打包的出口

      2. 编写 webpack 脚本

      在 package.json 中，创建一个执行脚本。"build": "webpack"

      3. 测试

      这样就可以执行npm run build来执行 webpack 打包了。
      发现会报如下错误：

      原因是: webpack 只能打包 js 文件. 对于后缀名为 vue 的文件不能打包. 如何解决 呢?

    通过 vue-loader 来打包 vue 文件

    四. 使用 vue-loader 打包 vue 文件

    vue-loader 官网

    1. 安装 vue-loader

    npm install -D vue-loader vue-templete-compiler

    2. 安装 css-loader

    npm install -D css-loader

    3. webpack 配置

    需要添加一个插件

    // 使用 node 的 path 模块
const path = require('path')

// 引入 vue-loader 插件
const VueLoaderPlugin = require('vue-loader/lib/plugin')

module.exports = {
    // 打包模式
    mode: 'development',
    // 打包的入口
    entry: './src/main.js',
    // 打包的出口
    output: {
        filename: 'bundle.js',
        // resolve? __dirname??
        path: path.resolve(__dirname, 'dist')
    },
    // 打包规则
    module: {
        rules: [
            {
                test: /\.vue$/,
                loader: 'vue-loader'
            }
        ]
    },
    plugins: [
        // 引入 vue-loader 插件
        new VueLoaderPlugin()
    ],
    resolve: {
        alias: {
            'vue': 'vue/dist/vue.js'
        }
    }
}

    4. 重新打包测试

    重新打包，会在 dist 目录下生成 bundle.js

    在 index.html 中引入 bundle.js 测试，发现报错

    报错原因

    Vue 会打包生成三个文件:

    • runtime only 的文件 vue.common.js
    • compiler only 的文件 compiler.js
    • runtime + compiler 的文件 vue.js

    而默认导出的是 vue.common.js, 如何解决呢?

    解决

    在 webpack 中，添加别名的配置

    resolve: {
    alias: {
        'vue': 'vue/dist/vue.js'
    }
}

    重新打包测试。正常出现this is App字样。

    5. 小结

    小结

    1. webpack 本身只能打包 js 文件，如果要打包其他文件就需要借助于 loader
    2. loader 起始就是专门用于打包特定文件的处理程序

    五. 其他常用 loader

    1. 打包图片文件

    1） file-loader

    file-loader 官方文档

    file-loader：将文件复制到对应的路径，并返回文件名

    i 安装

    npm install -D file-loader

    ii 配置

    修改 webpack 配置,添加一条规则

    {
    // 图片文件用 file-loader 打包
    test: /\.(jpg|jpeg|png|svg)$/,
    loader: 'file-loader',
    options: {
        // name占位符表示使用员文件名称
        // ext占位符表示使用原文件扩展名
        name: '[name].[ext]'
    }
}

    iii 测试

    在 src 目录下创建 assets 目录,存放静态资源文件。在 images 下放一张图片

    在 App.vue 中导入图片，重新打包，会在 dist 目录下生成我们需要的图片

    2）url-loader

    url-loader：功能类似于file-loader，但是在文件大小（单位 byte）低于指定的限制时，可以返回一个 DataURL。

    好处是：直接将小图片以 base64 打包在 js 中，减少 Http 请求的次数，提高访问效率

    i 安装

    npm install -D url-loader

    ii 配置

    {
    test: /\.(jpg|jpeg|png|svg)$/,
    loader: 'url-loader',
    options: {
        name: '[name].[ext]',
        limit: 4096
    }                
}

    iii 测试

    复制两张小的 svg 图片到 assets 目录下. 并在 App.vue 中导入

    // 导入小图片，测试 url-loader
import './assets/images/test.svg'

    结果：dist 目录下没有 test.svg 文件，在 bundle.js 追中发现 test.svg 直接打包到 js 中了。

    2. 打包 css 文件

    css-loader官方文档

    1）安装

    # css-loader 在安装 vue-loader 时已经安装，可以通过`package.json`确认
# style-loader
npm install -D style-loader

    2）配置

    {
    test: /\.css$/,
    // 配置两个 loader 打包 css 文件
    // 顺序从右到左，从下到上
    use: ['style-loader', 'css-loader']
}

    注意

    这里的顺序是有讲究的，从右到左，从下到上

    3）测试

    在 styles 目录下创建 reset.css 和 test.css 文件, 编写如下内容

    /* reset.css */
* {
    margin: 0;
    padding: 0;
}

body {
    background-color: red;
}

/* test.css */
/* css-loader 合并文件；
   style-loader 把样式挂载到 head 标签下 */
@import './reset.css'

    在 App.vue 中导入 test.css

    // 导入 css
import './assets/styles/test.css'

    重新打包，测试样式生效

    4）作用

    • css-loader：解决文件之间的依赖关系，把所有 css 文件打包成一个文件
    • style-loader：将 css-loader 打包完成后生成的文件挂载到页面 head 标签的 style 中

    3. 打包 stylus 文件

    目前, stylus 做为 node 项目普通使用的 css 预处理器被广泛的应用于 vue 项目中. 大家会发现大部分的 vue 项目中都会使用 stylus 来编写 css

    1）推荐 vscode 插件与配置

    • language-stylus：提供语法高亮和一些支持
    • Supremacy：自动格式化 stylus 的插件, 可以根据个人习惯或公司的要求, 定制 stylus 格式. 比如是否要;, :

    配置 settings.json

    // stylus 不使用 {}
"stylusSupremacy.insertBraces": false,
// stylus 不使用 ;
"stylusSupremacy.insertSemicolons": false,

    配置之后 stylus 的样子：

    body 
    background-color: red

    2）打包 stylus 文件

    i 安装 stylus-loader

    npm install -D stylus stylus-loader
    • stylus: 是 stylus 文件预处理程序, 作用是将 stylus 编译成 css 格式
    • stylus-loader: 加载 stylus 文件, 调用 stylus 预处理程序形成 css 文件

    ii 配置

    {
    test: /\.styl(us)?$/,
    use: ['style-loader', 'css-loader', 'stylus-loader']
}

    iii 测试

    在 styles 下编写 global.styl 文件, 编写如下内容, 这个也是响应式背景全屏的方案

    body
  // 背景图片
  background-image: url('..//images/todolist/bg.jpg')
  // 背景图片位置固定
  background-attachment: fixed
  // 背景不要重复
  background-repeat: no-repeat
  // 背景位置居中
  background-position: center center
  // 背景覆盖整个viewport
  background-size: cover
  // 当背景没有加载时的颜色
  background-color: #fff

    在 App.vue 中引入 global.styl

    // 导入 stylus 样式文件
import './assets/styles/global.stylus'

    3）处理 vue 文件中的 stylus

    vue-loader 关于 stylus 官方文档

    i 安装

    stylus、stylus-loader 已经安装

    ii 配置

    {
    test: /\.styl(us)?$/,
    use: ['vue-style-loader', 'css-loader', 'stylus-loader']
}
    • vue-style-loader: 是 vue-loader 自带的 style-loader, 在 style-loader 的基础上, 还可以处理 vue 文件中的样式

    iii 测试

    修改 App.vue 文字样式

    <style lang="stylus" scoped>
    h1 
        color: white
</style>

    使用vue-style-loader失败了，反而直接用style-loader成功了？

    六. 插件

    什么是插件？

    在某个时间点，自动执行的处理程序（类似于 vue 的生命周期函数）

    1. 使用 html-webpack-plugin 插件

    官方文档

    1）安装

    npm install -D html-webpack-plugin

    2）配置

    // 引入 html-webpack-plugin 插件
const HtmlWebpackPlugin = require('html-webpack-plugin')

plugins: [
    // 引入 vue-loader 插件
    new VueLoaderPlugin(),
    new HtmlWebpackPlugin()
],

    3）打包测试

    dist 目录下成功生成 index.html 文件，并且引入了 bundle.js，但是没有 #app 元素。

    4）指定模板

    new HtmlWebpackPlugin({
    template: './index.html'
})

    5）小结

    html-webpack-plugin 的作用：在打包结束时, 在 dist 目录下自动生成 index.html 文件, 并把打包好的 js 文件引入到 html 中

    2. 使用 clean-webpack-plugin

    1）安装

    npm install -D clean-webpack-plugin

    2）配置

    // 引入 clean-webpack-plugin 插件
const { CleanWebpackPlugin } = require('clean-webpack-plugin')

new CleanWebpackPlugin()

    3）测试

    每次打包前清理了 dist 文件夹

    3. 使用 autoprefixer 插件

    1）安装

    npm install -D postcss-loader autoprefixer@8.0.0

    2）配置

    {
    test: /\.styl(us)?$/,
    use: ['style-loader', 'css-loader', 'postcss-loader', 'stylus-loader']
}
# 新建 postcss.config.js 配置文件
module.exports = {
    plugins: [require('autoprefixer')]
}

    5）问题

    <template>
    <div>
        <h1>TodoList</h1>
        <input type="text" v-model="content" />
        <button @click="addTodo">添加</button>
        <ul>
            <li v-for="item of todoData" :key="item.id">{{ item }}</li>
        </ul>
    </div>
</template>

<script>
export default {
    name: 'App',
    data() {
        return {
            todoData: ['todo1', 'todo2', 'todo3'],
            content: ''
        }
    },
    methods: {
        addTodo() {
            if (this.content === '') return
            this.todoData.push(this.content)
            this.content = ''
        }
    }
}
</script>

    2. HRM-热模块替换

    官方文档

    Hot Module Replacement 或 HMR 允许在运行时更新各种模块，而无需进行完全刷新。

    注意热模块替换不适用于生成环境。

    1）启用

    i 配置 devServer

    devServer: {
    // 指定服务器根目录
    contentBase: './dist',
    // 自动打开浏览器
    open: true,
    // 启用热模块替换
    hot: true
}

    ii 配置插件

    // 引入 webpack
const webpack = require('webpack')

plugins: [
    // 引入 vue-loader 插件
    new VueLoaderPlugin(),
    // 自动生成 index.html 插件
    new HtmlWebpackPlugin({
        template: './index.html'
    }),
    // 打包前自动清理插件
    new CleanWebpackPlugin(),
    // 热模块替换插件
    new webpack.HotModuleReplacementPlugin()
]

    2）测试

    增加 todo 后，即使再修改然后重载，增加的 todo 还在。

    测试

    八. 生产环境

    • 开发环境（development）构建目标：实时重新加载、热模块替换、Source Map 和 localhost server
    • 生产环境（production）构建目标：更小的 bundle，更轻量的 source map，更优化的资源和加载时间

    所以分开两个配置文件

    • webpack.dev.js：用于开发环境
    • webpack.prod.js：用于生产环境

    webpack.base.js

    // 使用 node 的 path 模块
const path = require('path')
// 引入 vue-loader 插件
const VueLoaderPlugin = require('vue-loader/lib/plugin')
// 引入 html-webpack-plugin 插件
const HtmlWebpackPlugin = require('html-webpack-plugin')
// 引入 clean-webpack-plugin 插件
const { CleanWebpackPlugin } = require('clean-webpack-plugin')

module.exports = {
    // 打包的入口
    entry: './src/main.js', 
    // 打包的出口
    output: {
        filename: 'app.js',
        // resolve? __dirname??
        path: path.resolve(__dirname, 'dist')
    },
    // 插件
    plugins: [
        new VueLoaderPlugin(),
        new HtmlWebpackPlugin({
            template: './index.html'
        }),
        new CleanWebpackPlugin(),
    ],
    // 打包规则
    module: {
        rules: [
            {
                // .vue 文件用 vue-loader 打包
                test: /\.vue$/,
                loader: 'vue-loader'
            },{
                test: /\.(jpg|jpeg|png|svg)$/,
                loader: 'url-loader',
                options: {
                    name: '[name].[ext]',
                    limit: 4096
                }                
            },{
                test: /\.css$/,
                // 配置两个 loader 打包 css 文件
                // 顺序从右到左，从下到上
                use: ['style-loader', 'css-loader']
            },{
                test: /\.styl(us)?$/,
                use: ['style-loader', 'css-loader', 'postcss-loader', 'stylus-loader']
            }
        ]
    },
    resolve: {
        alias: {
            // 指定使用 vue.js，因为默认导出的是 runtime 包
            'vue': 'vue/dist/vue.js'
        }
    }
}

    webpack.dev.js 

    // 引入 webpack
const webpack = require('webpack')
