    ZooKeeper学习(二)

    高并发 ZooKeeper

    ZooKeeper paxos、ZAB、watch、API、callback（reactive）。

    ZooKeeper 作为一个分布式协调服务，具有扩展性、可靠性、时序性、快速的特征。

    1. 扩展性

    ZooKeeper 框架架构有 Leader、Follower、Observer 三类角色。ZooKeeper 提供读写分离，只有 Leader 才能写，只有 Follower 才能选举。可以在配置文件指定 Observer， Observer 只能读取，不能参与选举。 server.4=node04:2888:3888:observer。Observer 是为了放大查询能力，以便更加充分的发挥 ZooKeeper 的优势。

    2. 可靠性

    1. 快速恢复的 Leader
    2. paxos
    3. zab

    3. ZAB 协议

    Zookeeper 是通过 Zab 协议（原子广播协议）来保证分布式事务的最终一致性。ZAB 协议作用在 ZooKeeper 的状态为可用状态时，即 Leader 正常时。

    原子：要么成功要么失败，没有中间状态。（队列+两步操作）

    广播：分布式多节点，过半通过。

    比如客户端发起一个写操作的流程：

    1. 客户端向 Follower 发起 create 操作
    2. Follower 将请求转至 Leader
    3. Leader 向所有 Follower 广播，让 Follower 写 log
    4. 当有超过一半 Follower 同意，Leader 立即向所有 Follower 发送 write 指令
    5. Leader 通知被请求的 Follower create 完成
    6. Follower 回复客户端操作完成

    zk 的 Leader 会和每个 Follower 维护一个队列，从而保证数据的最终一致性和操作的原子性。

    4. 选举流程

    当 ZooKeeper 集群中 Leader 故障时，ZooKeeper 能快速推举出新 Leader，是如何实现的呢？

    1. 每个节点会有一个 myid 和最大的事务id（Zxid）
    2. 3888 端口就是选举端口，该集群中任何节点可以通过该端口和集群中其他节点两两通讯
    3. 所以只要集群中有任何节点发起投票，都会触发准 Leader 发起自己的投票
    4. 推选制度：先比较 Zxid，如果 Zxid 相同，再比较 myid

    jps 是什么命令？

    5. Watch

    ZooKeeper 提供的是分布式协调服务，协调是指协调客户端之间的行为。比客户端 A 和客户端 B 要进行通讯，需要知道互相之间服务是否正常，传统的方法可以通过心跳来实现，但是心跳总会有一个间隔，肯定或多或少会有时效性的问题。那么此时可以考虑用 ZooKeeper 来实现。

    1. ZooKeeper 和每个客户端连接都会有一个 session，并且 ZooKeeper 可以建立临时节点，如果客户端连接断了，呢么 session 到了超时时间后将随之消失，该客户端创建的临时节点也消失
    2. 所以 A 可以 create 一个临时节点，比如 /ooxx/a
    3. B get /ooxx/a
    4. B watch /ooxx/a
    5. ZooKeeper 有事件机制，如 create、delete、change、children 等
    6. 所以当 A 挂了的时候，B 立马会收到事件的 callback，就知道 A 出问题了

    总的来说，时效性更高，方向性是主动通知

    6. API 操作

    <dependencies>
    <dependency>
        <groupId>org.apache.zookeeper</groupId>
        <artifactId>zookeeper</artifactId>
        <version>3.4.14</version>
    </dependency>
</dependencies>
    public class Demo01 {
    public static void main(String[] args) throws IOException, InterruptedException, KeeperException {

        final CountDownLatch cd = new CountDownLatch(1);

        // zk 是 session 概念的，没有连接池的概念，即一个连接一个 session
        // zk 的 Watch 有两类。第一类：new zk 时传入的 watch，这个 watch session 级别的和 node、path 没有关系
        final ZooKeeper zooKeeper = new ZooKeeper("192.168.238.66:2181,192.168.238.68:2181,192.168.238.70:2181,192.168.238.72:2181",
                // session 超时时间
                80000, new Watcher() {
            public void process(WatchedEvent event) {
                Event.KeeperState state = event.getState();
                Event.EventType type = event.getType();
                System.out.println("event: " + event.toString());

                switch (state) {
                    case Unknown:
                        break;
                    case Disconnected:
                        break;
                    case NoSyncConnected:
                        break;
                    case SyncConnected:
                        cd.countDown();
                        System.out.println("SyncConnected......");
                        break;
                    case AuthFailed:
                        break;
                    case ConnectedReadOnly:
                        break;
                    case SaslAuthenticated:
                        break;
                    case Expired:
                        break;
                }

                switch (type) {
                    case None:
                        break;
                    case NodeCreated:
                        break;
                    case NodeDeleted:
                        break;
                    case NodeDataChanged:
                        break;
                    case NodeChildrenChanged:
                        break;
                }
            }
        });

        cd.await();
        ZooKeeper.States state = zooKeeper.getState();
        switch (state) {
            case CONNECTING:
                System.out.println("ing......");
                break;
            case ASSOCIATING:
                break;
            case CONNECTED:
                System.out.println("ed........");
                break;
            case CONNECTEDREADONLY:
                break;
            case CLOSED:
                break;
            case AUTH_FAILED:
                break;
            case NOT_CONNECTED:
                break;
        }

        // 新增节点
        // watch 的注册只发生在读类型调用，比如 get、exists；写是产生事件
        String pathName = zooKeeper.create("/111", "old data".getBytes(), ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.EPHEMERAL);

        // 查询节点
        final Stat stat = new Stat(); // 元数据信息
        byte[] data = zooKeeper.getData("/111", new Watcher() {
            public void process(WatchedEvent event) {
                System.out.println("getDate event:" + event.toString());

                try {
                    // 处理完成继续注册 watch，下次就该就会再进来
                    // true：注册 default watch，就是创建 zk 时的 watch
                    zooKeeper.getData("/111", this, stat);
                } catch (KeeperException e) {
                    e.printStackTrace();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, stat);
        System.out.println("节点 /111 数据：" + new String(data));

        // 修改节点
        // 会触发回调
        Stat stat1 = zooKeeper.setData("/111", "new data".getBytes(), 0);
        // 再次修改不会触发回调，watch 是一次性的
        Stat stat2 = zooKeeper.setData("/111", "new data01".getBytes(), stat1.getVersion());

        // 异步操作
        System.out.println("============sync start=====================");
        zooKeeper.getData("/111", false, new AsyncCallback.DataCallback() {
            public void processResult(int rc, String path, Object ctx, byte[] data, Stat stat) {
                System.out.println("============sync callback=====================");
                System.out.println(new String(data));
                System.out.println(ctx.toString());
            }
        }, "abc");
        System.out.println("============sync end=====================");

        // 当连接到的 zk 节点挂了时，会自动切换，并且 sessionId 不会变
        Thread.sleep(22222222);
    }
}
