    第 13 章 MySQL 数据库与 JDBC 编程

    13.1 JDBC 基础

    JDBC 为数据库开发提供了标准的 API，所以使用 JDBC 开发的数据库应用可以跨平台运行，而且可以跨数据库（如果全部使用标准的 SQL）。

    13.1.1 JDBC 简介

    Java 各种跨平台特性都采用相似的结构，因为他们都需要让相同的程序在不同的平台上运行，所以都需要中间的转换程序（为了实现 Java 程序的跨平台性，Java 为不同的操作系统提供了不同的虚拟机）

    同样，为了使 JDBC 程序可以跨平台，则需要不同的数据库厂商提供相应的驱动程序。

    13.1.2 JDBC 驱动程序

    数据库驱动程序是 JDBC 程序和数据库之间的转换层，它负责将 JDBC 调用映射成特定的数据库调用。

    JDBC 驱动通常有如下 4 种类型：

    • JDBC-ODBC 桥：将 JDBC API 映射到 ODBC API。Java 8 已经删除。
    • JDBC API 映射成数据库特定的客户端 API。驱动包含特定数据库的本地代码
    • 支持三层结构的 JDBC 访问方式，主要用于 Applet 阶段
    • 纯 Java 的，直接与数据库实例交互。是智能的，它知道数据库使用的底层协议。

    1. 下载： https://dev.mysql.com/downloads/windows/installer/8.0.html
    2. 安装

    13.2.2 关系数据库基本概念和 MySQL 基本指令

    DBMS 是所有数据的知识库，它负责管理数据的存储、安全、一致性、并发、恢复和访问等操作。

    DBMS 有一个数据字典(有时也被称为系统表) .用于存储它拥有的每个事务的相关信息，例如名字、结构、位置和类型，这种关于数据的数据也被称为元数据（metadata ）。

    MySQL 数据库的一个实例（Server Instance）可以同时包含多个数据库，

    -- 连接到指定实例
mysql -h localhost -u root -p --default-character-set=utf8

-- 查看当前实例下包含多少个数据库
show databases;
-- 创建数据库
create database if not exists test;
-- 删除数据库
drop database test;
-- 进入指定数据库
use test;
-- 显示所有数据表
show tables;
-- 查看表信息
desc tablename;

    MySQL 通常支持如下两种存储机制：

    • MyISAM：早期默认的存储机制，对事务的支持不够好
    • InnoDB：事务安全的存储机制。系统默认启动 InnoDB 存储机制，如果不想用，可以使用 skip-innodb 选项

    建表时指定存储机制：

    • ENGINE=MyISAM
    • ENGINE=InnoDB

    13.2.3 SQL 语句基础

    SQL 语句分类：

    • 查询语句
    • DML（Data Manipulation Language）：insert、update、delete
    • DDL（Data Definition Language）：create、alter、drop、truncate
    • DCL（Data Control Language）：grant、revoke
    • 事务控制语句：commit、rollback、savepoint

    标识符：

    • 字母开头
    • # _ $ 三个特殊符号
    • 不可使用数据库系统关键字、保留字
    • 同一个模式（外模式）下对象不应该同名

    13.2.4 DDL 语句

    创建表

    create table test
(
    test_id int,
    test_price decimal,
    test_name varchar(255) default 'xxx',
    -- 大文本类型
    test_desc text,
    -- 图片
    test_img blob,
    test_date datetime
);

    MySQL 支持的数据类型：

    列类型 说明
    tinyint/smallint/dediumint/int/bigint 1/2/3/4/8字节整数
    float/double 单精度、双精度浮点类型
    decimal 精确小数类型
    date/time/datetime 日期/时间/日期时间类型
    timestamp 时间戳
    year 年
    char 定长字符串类型
    binary 定长二进制字符串类型
    varbinary 变长
    tinyblob/blob/mediumblob/longblob 1/2/3/4字节的二进制大对象，可用于存储图片、音乐等二进制数据，分别可存储：255B/64KB/16MB/4GB
    tinytext/text/mediumtext/longtext 同时，文本对象
    enum(‘value1’,’value2’,…) 枚举
    set(‘value1’,’value2’,…) 集合

    子查询建表：

    create table hehe
as
select * from test_table;

    修改表结构

    增加列、修改列、删除列、重命名列

    -- 增加列
alter table hehe add(
    aaa varchar(255) default 'xxx',
    bbb varchar(255)
);

-- 修改列
alter table hehe modify aaa text, modify bbb int;

-- 删除列
alter table drop aaa;

-- 重命名（MySQL）
alter table hehe rename to wawa;
alter table wawa change bbb ddd int;

    删除表

    drop table wawa;

    truncate 表

    truncate wawa;

    truncate 比 delete 性能更好。

    对于 MySQL：如果使用非 InnoDB 存储机制，truncate 比 delete 速度要快；如果使用 InnoDB，在 MySQL 5.0.3 之前和 delete 完全一样，在 5.0.3 之后，truncate 效率更高，但如果该表被外键约束所参照，又变得和 delete 一样。

    13.2.5 数据库约束

    • NOT NULL
    • UNIQUE：不可重复，但可以多个null，因为 null != null
    • PRIMARY KEY：不可重复并且不可为 null
    • FOREIGN KEY
    • CHECK(MySQL 不支持)

    MySQL 使用 information_schema 数据库里的 TABLE_CONSTRAINTS 表来保存约束信息。

    -- 建表时指定非空约束
create table hehe(
    hehe_id int not null,
    hehe_name varchar(255) default 'xyz' not null,
    hehe_gender varchar(2) null
);
-- alter 增加/取消非空约束
alter table hehe modify hehe_gender varchar(2) not null;
alter table hehe modify hehe_name varchar(2) null;

-- 唯一约束
create table unique_test(
    test_id int not null,
    -- 使用列级约束语法建立唯一约束
    test_name varchar(255) unique,
    test_gender varchar(2),
    test_pass varchar(255),
    -- 使用表级约束语法建立唯一约束，并指定约束名称
    constraint test_uk unique(test_gender),
    -- 组合列唯一约束
    constraint test1_uk unique(test_name,test_pass)
);
-- alter 增加/取消唯一约束
alter table unique_test add unique(test_name,test_pass);
alter table unique_test modify test_name varchar(255) unique;
alter table unique_test drop index test1_uk;

-- 主键约束
create table primary_test(
    test_id int auto_increment primary key,
    test_name varchar(255)
);
create table primary_test2(
    test_id int not null,
    test_name varchar(255),
    test_pass varchar(255),
    -- 虽然指定 test2_pk 名字，但无效
    constraint test2_pk primary key(test_id)
);
create table primary_test3(
    test_name varchar(255),
    test_pass varchar(255),
    -- 多列主键
    primary key(test_name, test_pass)
);

alter table primary_test3 add primary key(test_name, test_pass);
alter table primary_test3 modify test_name varchar(255) primary key;
alter table primary_test3 drop primary key;

-- 外键约束
create table teacher_table(
    teacher_id int auto_increment,
    teacher_name varchar(255),
    primary key(teacher_id)
);
create table student_table(
    student_id int auto_increment primary key,
    student_name varchar(255),
    -- 列级语法不会生效，仅仅是为了兼容 sql 语法
    java_teacher int references teacher_table(teacher_id)
);

create table teacher_table1(
    teacher_id int auto_increment,
    teacher_name varchar(255),
    primary key(teacher_id)
);
create table student_table1(
    student_id int auto_increment primary key,
    student_name varchar(255),
    java_teacher int,
    foreign key(java_teacher) references teacher_table1(teacher_id)
        on delete cascade -- 也可 on delete set null
);

create table teacher_table3(
    teacher_name varchar(255),
    teacher_pass varchar(255),
    primary key(teacher_name,teacher_pass)
);
create table student_table3(
    student_id int auto_increment primary key,
    student_name varchar(255),
    java_teacher_name varchar(255),
    java_teacher_pass varchar(255),
    constraint student_teacher_fk foreign key(java_teacher_name,java_teacher_pass) 
        references teacher_table3(teacher_name,teacher_pass)
);

alter table student_table3 drop foreign key student_teacher_fk;
alter table student_table3 add foreign key(java_teacher_name, java_teacher_pass)
    references teacher_table3(teacher_name,teacher_pass);

-- check 约束
create table check_test(
    emp_id int primary key auto_increment,
    emp_name varchar(255),
    emp_salary decimal,
    check(emp_salary>0)
);

    13.2.6 索引

    创建索引的唯一作用就是加速对表的查询，索引通过使用快速路径访问方法来快速定位数据，从而减少了磁盘的 I/O。

    MySQL 使用 information_schema 数据库里的 STATISTICS 表来保存索引。

    创建索引的方式：

    • 自动：主键约束、唯一约束、外键约束
    • 手动：create index

    删除索引的方式：

    • 自动：数据表被删除
    • 手动：drop index

    坏处：

    • 数据表中记录被添加、删除、修改时需要维护索引
    • 存储索引信息需要一定的磁盘空间
    create index emp_last_name_idx on employees(first_name,last_name);
drop index emp_last_name_idx on exployees;

    13.2.7 视图

    一个或多个数据表的逻辑显示，本身不能存储数据。

    • 限制对数据的访问
    • 使复杂的查询变简单
    • 提供了数据的独立性
    • 提供了对相同数据的不同显示
    create or replace view view_test as
select teacher_name,teacher_pass from teacher_table3
-- 指定不允许修改该视图的数据
with check option;

drop view view_test;

    13.2.10 数据库函数

    MySQL 单行函数特征：

    • 参数可以是变量、常量或数据列。可以多个参数，但只返回一个值。
    • 对每行单独起作用，每行返回一个结果
    • 可以改变参数的数据类型，支持嵌套使用。

    分类：

    • 数值型
    • 字符型
    • 日期时间型
    • 转换函数
    • 其他
      • 位函数
      • 流程控制函数
      • 加密解密函数
      • 信息函数
    select char_length(teacher_name) from teacher_table3;
select sin(char_length(teacher_name)) from teacher_table3;
select date_add('1998-01-02', interval 2 month);
select adddate('1998-01-02', 3);
select curdate();
select curtime();
select md5("testing");

select ifnull(student_name, '没有名字')from student_table3;
-- 名字为张三则返回 null
select nullif(student_name, '张三')from student_table3;
select if(isnull(student_name),'没有名字','有名字') from student_table3;

select student_name, case java_teacher_name
  when 1 then 'Java老师'
  when 2 then 'Ruby老师'
  else '其他老师'
  end teacher
  from student_table3;

select student_name, case
  when student_id<=3 then '初级班'
  when student_id<=6 then '中级班'
  else '高级班'
  end class
  from student_table3;

    虽然 MySQL 提供了一些函数，但通常不推荐在 Java 程序中使用特定数据库的函数。

    3.2.11 分组和组函数

    select count(*) from student_table;
select count(distinct java_teacher) from student_table;
select sum(student_id) from student_table;
select sum(20) from student_table;
select max(student_id) from student_table;
select sum(distinct 20) from student_table;
-- null 不会计算在内
select count(student_name) from student_table;
select avg(ifnull(java_teacher, 0)) from student_table;

    having 对组进行过滤，只有满足该条件表达式的分组才会被选出来。

    • 不能在 where 子句中过滤组， where 子句仅用于过滤行。过滤组必须使用 having 子句。
    • 不能在 where 子句中使用组函数， having 子句才可使用组函数。
    select * from student_table group by java_teacher 
  having count(*)>2;

    13.2.2 多表连接查询

                }
            } 
            else

        }
    }

    public void query(String sql) throws Exception {

                                                         ResultSet.CONCUR_UPDATABLE);
         ResultSet rs = pstmt.executeQuery()) {
        rs.last();
        int rowCount = rs.getRow();

    13.6 Java 的 RowSet

    RowSet 接口继承了 ResultSet 接口。RowSet 接口下包含 JdbcRowSet、CachedRowSet、FilteredRowSet、JoinRowSet 和 WebRowSet 常用子接口。除了 JdbcRowSet，其余 4 个都是离线的 RowSet。

    与 ResultSet 相比， RowSet 默认是可滚动、可更新、可序列化的结果集，而且作为JavaBean 使用，因此能方便地在网络上传输，用于同步两端的数据。

        rs.close();
        stmt.close();
        conn.close();
        return cachedRs;
    }

    public static void main(String[] args) throws Exception {
        CachedRowSetTest ct = new CachedRowSetTest();
        ct.initParam("./src/unit13_jdbc/mysql.ini");
        CachedRowSet rs = ct.query("select * from student_table");
        rs.afterLast();
        while (rs.previous())
        {
            System.out.println(rs.getString(1) + "\t" + rs.getString(2) + "\t" + rs.getString(3));
            if (rs.getInt("student_id") == 2)
            {
                rs.updateString("student_name", "孙悟空");
                rs.updateRow();
            }
        }
        // 重新获取数据库连接
        Connection conn = DriverManager.getConnection(url, user, pass);
        conn.setAutoCommit(false);
        // 把对 RowSet 所做的修改同步到底层数据库

    }

}

    13.6.3 离线 RowSet 的查询分页

    由于 CachedRowSet 会将数据记录直接装载到内存中，因此如果 SQL 查询返回的记录过大，CachedRowSet 将会占用大量的内存，在某些极端的情况下，它甚至会直接导致内存溢出。

    • populate(ResultSet rs, int startRow)
    • setPageSize(int pageSize)
    • previousPage()：在底层 ResultSet 可用的情况下，读取上一页记录
    • nextPage()
    public CachedRowSet query(String sql, int pageSize, int page) throws Exception
{
    try(
        Connection conn = DriverManager.getConnection(url, user, pass);
        Statement stmt = conn.createStatement(ResultSet.TYPE_SCROLL_INSENSITIVE,ResultSet.CONCUR_UPDATABLE);
        ResultSet rs = stmt.executeQuery(sql))
    {
        RowSetFactory factory = RowSetProvider.newFactory();
        CachedRowSet cachedRs = factory.createCachedRowSet();
        // 设置煤业显示 pagesize 条记录
        cachedRs.setPageSize(pageSize);
        // 从指定位置开始装
        cachedRs.populate(rs, (page - 1) * pageSize + 1);
        //            cachedRs.populate(rs);

        cachedRs.nextPage();
        // 也不行
        // cachedRs.previousPage();
        return cachedRs;
    }
}

    13.7 事务处理

    13.7.1 事务的概念和 MySQL 事务支持

    事务是由一步或几步数据库操作序列组成的逻辑执行单元，这系列操作要么全部执行，要么全部放弃执行。

    • 原子性（Atomicity）：事务是应用中最小的执行单位
    • 一致性（Consistency）：事务执行的结果， 必须使数据库从一个一致性状态， 变到另一个一致性状态。一致性通过原子性来保证。例如 A 向 B 转账。
    • 隔离性（Isolation）：各个事务的执行互不干扰
    • 持续性（Durability）：持久性，事务一旦提交，改变是永久性的。

    事务组成：

    • 一组 DML 语句
    • 一条 DDL 语句
    • 一条 DCL 语句

    DDL 和 DDL 语句最多只能一条，因为 DDL 和 DCL 语句都会导致事务立即提交。

    事务提交：

    • 显示提交：commit
    • 自动提交：执行 DDL 或 DCL 语句，或程序正常退出

    事务回滚：

    • 显示回滚：rollback
    • 自动回滚：系统错误或者强行退出

    MySQL 默认关闭事务（即打开自动提交）

    -- 关闭自动提交，即开启事务
SET AUTOCOMMIT = 0
-- 临时开始事务
begin/start transactin
insert into student_table values(null,'xx',1);
insert into student_table values(null,'yy',1);
insert into student_table values(null,'zz',1);
select * from student_table s where s.student_name in ('xx','yy','zz');
-- 回滚
rollback;
select * from student_table s where s.student_name in ('xx','yy','zz');

start transaction;
insert into student_table values(null,'xx',1);
savepoint a;
insert into student_table values(null,'yy',1);
select * from student_table s where s.student_name in ('xx','yy');
-- rollback\commit 会结束事务，但 rollback to a 不会
rollback to a;
select * from student_table s where s.student_name in (xx','yy');
commit;

    public class TransactionTest {

    private static String driver;
    private static String url;

    {

        props.load(new FileInputStream(paramFile));

        url = props.getProperty("url");

    public void insertInTransaction(String[] sqls) throws Exception
    {
        try(
            Connection conn = DriverManager.getConnection(url, user, pass))
        {
            // 关闭自动提交，开启事务
            conn.setAutoCommit(false);
            try(
                Statement stmt = conn.createStatement())
            {
                for (String sql : sqls)
                {
                    stmt.executeUpdate(sql);
                }
            }
            conn.commit();
        }
    }

                // 违反外键约束
                "insert into student_table(student_name,java_teacher) values('ccc',5)"
        };
        tt.insertInTransaction(sqls);
    }

    public void insertBatch(String[] sqls) throws Exception {
    try (Connection conn = DriverManager.getConnection(url, user, pass)) {
        // 保存当前的模式
        boolean autoCommit = conn.getAutoCommit();
        // 关闭自动提交,开启事务
        conn.setAutoCommit(false);
        try (Statement stmt = conn.createStatement()) {
            // 循环插入 sql
            for (String sql : sqls) {
                stmt.addBatch(sql);
            }
            // 同时提交所有的 SQL
            stmt.executeLargeBatch();
            conn.commit();
            // 恢复原来的提交模式
            conn.setAutoCommit(autoCommit);
        }
    }
}

    public void printResultSet(ResultSet rs) throws SQLException
    {
        ResultSetMetaData rsmd = rs.getMetaData();
        for (int i = 0; i < rsmd.getColumnCount(); i++)
        {
            System.out.print(rsmd.getColumnName(i + 1) + "\t");
        }
        System.out.print("\n");
        // 打印 ResultSet 里的全部数据
        while (rs.next())
        {
            for (int i=0; i < rsmd.getColumnCount(); i++)
            {
                System.out.print(rs.getString(i + 1) + "\t");
            }
            System.out.print("\n");
        }
        rs.close();
    }

    13.9 使用连接池管理连接

    当应用程序启动时，系统主动建立足够的数据库连接，并将这些连接组成一个连接池。每次应用程序请求数据库连接时，无须重新打开连接，而是从连接池中取出己有的连接使用，使用完后不再关闭数据库连接，而是直接将连接归还给连接池。通过使用连接池，将大大提高程序的运行效率。

    数据库连接池常用参数：

    • 初始连接数
    • 最大连接属
    • 最小连接数
    • 每次增加的容量

    JDBC 数据库连接池使用 javax.sql.DataSource 来表示。DataSource 只是一个接口。通常由商用服务器(如WebLogic 、WebSphere) 等提供实现，也有一些开源组织提供实现(如 DBCP 和 C3PO 等)。
