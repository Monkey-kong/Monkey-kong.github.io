    静态代理到动态代理到 AOP。

    模式定义：为其他对象提供一种代理以控制对这个对象的访问。

    模式引入：我们有一辆坦克，坦克有 move 行为，现在接到一个需求，我们需要测试 move 方法的性能，并在 move 执行前后记录日志。

    1. 直接修改坦克源码

    这种方式可能是最容易想到也最容易实现的。

    public void move() {
    long start = System.currentTimeMillis();
    System.out.println("Tank moving claclacla...");
    try {
        // 模拟坦克 move
        Thread.sleep(new Random().nextInt(10000));
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    long end = System.currentTimeMillis();
    System.out.println(end - start);
}

    但是，我们面临一个问题，tank 的源码可能是第三方代码不允许修改。

    2. 使用继承

    class TimeTank extends Tank {
    @Override
    public void move() {
        long start = System.currentTimeMillis();
        super.move();
        long end = System.currentTimeMillis();
        System.out.println(end - start);
    }
}

    使用继承的确可以实现这个功能。但是这样我们代码耦合度和 Tank 太高，因为性能测试、日志记录等功能可能不仅仅坦克的 move 需要，很可能其他 Movable 对象也是需要的，比如卡车的 move，如果都使用继承实现，我们系统肯定会类爆炸。

    3. 使用静态代理

    我们新建一个代理，这个代理和 Tank 实现相同的 Movable 接口，并且持有一个 Movable 成员变量，表示需要代理的对象。这样所有的 Movable 类型的对象都可以传给这个代理来处理。

    class TankTimeProxy implements Movable {

    // 代理 Movable 对象
    Movable movable;

    public TankTimeProxy(Movable movable) {
        this.movable = movable;
    }

    @Override
    public void move() {
        // 代理插入的逻辑
        long start = System.currentTimeMillis();
        // 被代理对象原逻辑
        movable.move();
        // 代理插入的逻辑
        long end = System.currentTimeMillis();
        System.out.println(end - start);
    }
}

    这样我们使用时，只需要把需要代理的对象传给代理，就可以像使用原对象一样了。并且由于代理中持有的成员变量是 Movable 类型，所以可以代理所有的 Movable 对象。这就是我们常说的依赖倒置原则，即依赖抽象而并不是具体或者说面向接口编程。

    public static void main(String[] args) {
    // 把 tank 做为代理对象传给 TankTimeProxy 代理
    // 然后把 TankTimeProxy 作为代理对象传给 TankLogProxy 代理然后执行
    new TankLogProxy(new TankTimeProxy(new Tank())).move();
}

    这时程序看上去非常完美了，但是其实我们现在的代理其实只能代理 Movable 类型的对象，而没法代理其他类型对象，如果需要代理其他类型对象就需要使用动态代理了。所谓的动态代理就是我们不再需要手动写代理类了，系统会自动生成代理类，这个代理类我们看不到（可以配置系统参数 saveGeneratedFiles 保存文件然后反编译看到）。

    4. 使用 JDK 自带的动态代理

    jdk Proxy 类提供了动态代理的功能，但是 jdk 代理是基于接口的，也就是说被代理的对象必须实现某个接口，否则没法代理。

    public static void main(String[] args) {
    Tank tank = new Tank();

    // 保存一下动态生成的那个代理类
    // System.getProperties().put("jdk.proxy.ProxyGenerator.saveGeneratedFiles", "true");
    System.getProperties().put("sun.misc.ProxyGenerator.saveGeneratedFiles", "true");

    // 动态生成代理对象
    Movable m = (Movable) Proxy.newProxyInstance(
        Tank.class.getClassLoader(), // 代理对象的 classLoader，和被代理对象一致即可
        new Class[]{Movable.class},  // 代理对象应该实现哪些接口
        new LogHandler(tank)); // 被代理对象那个方法被调用时，我们怎么做处理
    m.move();
}
    class LogHandler implements InvocationHandler {
    Tank tank;
    public LogHandler(Tank tank) {
        this.tank = tank;
    }
    /**
     * @param proxy 生成的代理对象（m）
     * @param method 代理的方法
     * @param args 方法参数
     */
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("method "  + method.getName() + " start...");
        // 调用被代理对象的方法（tank.move()）
        Object o = method.invoke(tank, args);
        System.out.println("method " + method.getName() + " end");
        return o;
    }
}

    5. JDK 动态代理实现原理

    5.1 JDK 代理类

    其实用上面代码就大致可以看出，它的实现原理应该和静态代理类似，只是这个代理类是运行时动态生成的。我设置了 sun.misc.ProxyGenerator.saveGeneratedFiles 属性，保存了这个代理类（**$Proxy0.class**）

    final class $Proxy0 extends Proxy implements Movable {
    private static Method m1;
    private static Method m4;
    private static Method m2;
    private static Method m3;
    private static Method m0;

    // 构造函数，就是我们外边传入的 LogHandler 对象
    public $Proxy0(InvocationHandler var1) throws  {
        // Proxy：this.h = h; 
        super(var1);
    }

    // 代理类的 move 方法
    public final void move() throws  {
        try {
            // 调用 LogHandler 的 move 方法
            super.h.invoke(this, m4, (Object[])null);
        } catch (RuntimeException | Error var2) {
            throw var2;
        } catch (Throwable var3) {
            throw new UndeclaredThrowableException(var3);
        }
    }

    static {
        try {
            m1 = Class.forName("java.lang.Object").getMethod("equals", Class.forName("java.lang.Object"));
            m4 = Class.forName("com.monkeykong.proxy.v05.Movable").getMethod("move");
            m2 = Class.forName("java.lang.Object").getMethod("toString");
            m3 = Class.forName("com.monkeykong.proxy.v05.Movable").getMethod("go");
            m0 = Class.forName("java.lang.Object").getMethod("hashCode");
        } catch (NoSuchMethodException var2) {
            throw new NoSuchMethodError(var2.getMessage());
        } catch (ClassNotFoundException var3) {
            throw new NoClassDefFoundError(var3.getMessage());
        }
    }
}

    5.2 ASM

    还有一个问题，我们发现 $Proxy0 这个类没有源文件就直接生成了对象给我们使用，这是怎么实现的呢？这是通过 ASM 实现的，一个字节码操作和分析框架。 ASM is an all purpose Java bytecode manipulation and analysis framework. ASM 官网： https://asm.ow2.io/

    比如我们使用 ASM api 直接生成一个 Comparable 类例子：

    public class ClassWriteTest {
    public static void main(String[] args) {
        ClassWriter cw = new ClassWriter(0);
        cw.visit(V1_5, ACC_PUBLIC + ACC_ABSTRACT + ACC_INTERFACE,
                "pkg/Comparable", null, "java/lang/Object",
                null);
        cw.visitField(ACC_PUBLIC + ACC_FINAL + ACC_STATIC, "LESS", "I",
                null, -1).visitEnd();
        cw.visitField(ACC_PUBLIC + ACC_FINAL + ACC_STATIC, "EQUAL", "I",
                null, 0).visitEnd();
        cw.visitField(ACC_PUBLIC + ACC_FINAL + ACC_STATIC, "GREATER", "I",
                null, 1).visitEnd();
        cw.visitMethod(ACC_PUBLIC + ACC_ABSTRACT, "compareTo",
                "(Ljava/lang/Object;)I", null, null).visitEnd();
        cw.visitEnd();
        byte[] b = cw.toByteArray();

        MyClassLoader myClassLoader = new MyClassLoader();
        Class c = myClassLoader.defineClass("pkg.Comparable", b);
        // 输出 compareTo，可以看出 pkg.Comparable 这个类已经存在于 JVM 中。
        System.out.println(c.getMethods()[0].getName());
    }
}

    6. 使用 cglib 的动态代理

    除了使用 JDK 自带的动态代理功能，cglib 也提供了动态代理的功能，不同的是 cglib 不需要被代理的对象实现什么接口，普通的类也可以被代理。cglib 的底层实现也是 ASM。

    public class Main {
    public static void main(String[] args) {
        Enhancer enhancer = new Enhancer();
        // 这里就可以看出，cglib 的代理类会继承被代理类
        enhancer.setSuperclass(Tank.class);
        enhancer.setCallback(new TimeMethodInterceptor());
        Tank tank = (Tank)enhancer.create();
        tank.move();
    }
}

class TimeMethodInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {
        // 所以生成的代理类是 Tank 类的一个子类。所以 final 类不能使用 cglib 实现动态代理
        // System.out.println(o.getClass().getSuperclass().getName());
        System.out.println("before");
        Object result = null;
        result = methodProxy.invokeSuper(o, objects);
        System.out.println("after");
        return result;
    }
}

    7. Spring AOP

    我们都知道 Spring 的核心之一就是 AOP，AOP 就是动态代理的典型应用。下面就写一下基本的例子。

    7.1 schema-based approach

    业务代码。

    public class Tank {
    public void move() {
        System.out.println("Tank moving claclacla...");
        try {
            Thread.sleep(new Random().nextInt(10000));
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

public class TimeProxy {
    public void before() {
        System.out.println("method start..." + System.currentTimeMillis());
    }
    public void after() {
        System.out.println("method end..." + System.currentTimeMillis());
    }
}

public class Main {
    public static void main(String[] args) {
        // 加载 Spring 容器
        ApplicationContext context = new ClassPathXmlApplicationContext("app.xml");
        // 从容器中取出一个 bean
        Tank tank = (Tank) context.getBean("tank");
        tank.move();
    }
}

    额，发现什么都没写啊，怎么就动态代理了？别急，Spring 的理念就是让你发现用了就跟没用一样，主要在 Spring 的配置。

    <bean id="tank" class="com.monkeykong.proxy.springaop.v1.Tank"/>
<bean id="timeProxy" class="com.monkeykong.proxy.springaop.v1.TimeProxy"/>
<aop:config>
    <!-- 切面：插什么。需要插入的内容 -->
    <aop:aspect id="myAspect" ref="timeProxy">
        <!-- 切点：插到哪里。需要被代理的方法 -->
        <aop:pointcut id="tankPointcut" expression="execution(* com.monkeykong.proxy.springaop.v1.Tank.move())"/>
        <!-- 通知：什么时候插 -->
        <aop:before pointcut-ref="tankPointcut" method="before"/>
        <aop:after pointcut-ref="tankPointcut" method="after"/>
    </aop:aspect>
</aop:config>

    7.2 @AspectJ annotation style

    使用注解配置：

    @Aspect // 插入什么
public class TimeProxy {
    // 这个方法什么时候插入（before），插入到哪里（move）
    @Before("execution(void com.monkeykong.proxy.springaop.v2.Tank.move())")
    public void before() {
        System.out.println("method start..." + System.currentTimeMillis());
    }
    @After("execution(void com.monkeykong.proxy.springaop.v2.Tank.move())")
    public void after() {
        System.out.println("method end..." + System.currentTimeMillis());
    }
}

    在 xml 中启用 aop：

    <bean id="tank" class="com.monkeykong.proxy.springaop.v2.Tank"/>
<bean id="timeProxy" class="com.monkeykong.proxy.springaop.v2.TimeProxy"/>

<!-- 启用 aop -->
<aop:aspectj-autoproxy/>
