第 14 章 注解。

注解被用来为程序元素（类、方法、成员变量等）设置元数据。注解不影响程序代码的执行，无论增加、删除注解，代码都始终如一的执行。

如果希望让程序中的注解在运行时起一定的作用，只有通过某种配套的工具对注解中的细腻进行访问和处理，访问和处理注解的工具统称 APT。

14.1.1 限定重写父类方法 @Override

14.1.2 Java 9 增强的 @Deprecated

Java 9 为 @Deprecated 注解增加了如下两个属性：

  • forRemoval：指定该 API 在将来是否会被删除
  • since

@Deprecated(since="9", forRemoval=true)

14.1.4 堆污染警告与 Java 9 增强的 @SafeVarargs

堆污染：把一个不带泛型的对象赋值给一个带泛型的变量时，往往就会发生这种堆污染

使用 @SafeVarargs 可以抑制该警告。

14.1.5 Java 8 的函数式接口与 @FunctionInterface

如果接口中只有一个抽象方法（可以包含多个默认方法或多个 static 方法），该接口就是函数式接口。

@FunctionInterface 就是用来指定某个接口必须是函数式接口。

14.2 JDK 的元注解

除了5个基本注解，Java 还提供了 6 个 Meta 注解，专门你用于修饰其他的注解的定义。

14.2.1 @Retention

指定被修饰的注解可以保留多长时间。

  • RententionPolicy.CLASS：注解记录在 class 文件中，当运行 Java 程序时， JVM 不可获取注解信息。这是默认值。
  • RententionPolicy.RUNTIME：注解记录在 calss 文件中，运行时 JVM 可以获取注解信息，程序可以通过反射获取该注解的信息。
  • RententionPolicy.SOURCE：注解只保留在源代码，编译器直接丢弃这种注解。

14.2.2 @Target

指定被修饰的注解能用于修饰哪些程序单元。

  • ElementType.ANNOTATION_TYPE：只能修饰注解
  • ElementType.CONSTRUCTOR：只能修饰构造器
  • ElementType.FIELD：只能修饰成员变量
  • ElementType.LOCAL_VARIABLE：只能修饰局部变量
  • ElementType.METHOD：只能修饰方法
  • ElementType.PACKAGE：只能修饰包定义
  • ElementType.PARAMETER：只能修饰参数
  • ElementType.TYPE：可以修饰类、接口、枚举

14.2.3 @Documented

指定注解类将被 javadoc 工具提取成文档。

14.2.4 @Inherited

指定被修饰的注解将具有继承性。如果某个类使用了@Xxx注解（定义该注解时使用了 @Inherited 修饰）修饰，则其子类将自动被@Xxx修饰。

14.3 自定义注解

14.3.1 定义注解

@interface 定义注解。

以无形参的方法形式来声明成员变量。

根据注解是否可以包含成员变量，可以把注解分为如下两类：

  • 标记注解：没有定义成员变量。这种注解仅利用自身存在与否来提供信息
  • 元数据注解：包含成员变量的注解，可以接收更多的元数据

14.3.2 提取注解信息

使用注解修饰了类、方法、成员变量等成员后，这些注解不会自己生效，必须由开发者提供相应的工具来提取注解信息。

AnnotatedElement 接口是所有程序元素如（Class、Method、Constructor等）的父接口，所以程序通过反射获取了某个类的 AnnotatedElement对象之后，就可以访问注解信息：

  • A getAnnotation(Class annotationClass)：返回该程序元素上存在的、指定类型的注解
  • A getDeclaredAnnotation(Class annotationClass)：获取直接修饰该程序元素、指定类型的注解。
  • Annotation[] getAnnotaions()：返回该程序元素上的所有注解
  • Annotation[] getDeclaredAnnotations()：返回直接修饰该程序元素的所有注解
  • boolean isAnnotationPresent(Class<? extends Annotation> annotationsClass)：判断该程序元素上是否存在指定类型的注解

  • A[] getAnnotationsByType(Class annotationClass)：获取修饰该程序元素、指定类型的多个注解。因为 Java 增加了重复注解

  • A[] getDeclaredAnnotationsByType(Class annotationClass)

为了获得程序中的程序元素(女口Class 、Method 等)，必须使用反射知识。

如果需要获取某个注解里的元数据，则可以将注解强制类型转换成所需的注解类型， 然后通过注解对象的抽象方法来访问这些元数据。

14.3.5 Java 8 新增的类型注解

Java 8 为 ElementType 枚举增加了 TYPE_PARAMETER、TYPE_USE 两个枚举值。@Target(ElementType.TYPE_USE) 定义的注解称为类型注解。

Java 8 以前，只能在定义各种程序元素（定义类、接口、方法、成员变量、…）时使用注解。

从Java 8 开始， 类型注解可以修饰在任何地方出现的类型。比如：

  • 创建对象
  • 类型转换
  • 使用 implements 实现接口
  • 使用 throws 声明抛出异常

14.4 编译时处理注解

Java 提供的 java.exe 工具有一个 -processor 选项，该选项可以指定一个注解处理器。

APT 可以根据源文件中的注解来生成额外的文件。APT 还会编译生成的源代码文件和原来的源文件，将他们一起生成 class 文件。
